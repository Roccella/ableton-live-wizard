# Protocol - Audit Loop

## Goal
- Diagnose the current loop before proposing changes.

## Audit Checklist
- Identify likely genre and confidence.
- Identify the main roles already present.
- Describe the loop's current energy and function.
- Identify what is missing or overloaded.
- Point out the next 3 useful moves in priority order.

## Output Shape
- `Genre and confidence`
- `Current roles`
- `What is working`
- `What is missing or weak`
- `Next 3 actions`

## Behavior Rules
- Prefer concrete observations over generic praise.
- If the genre is ambiguous, say what it is between.
- Do not jump into editing unless the user explicitly asked for it.
